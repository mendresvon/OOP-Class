## Context

本專案旨在將智慧型租借管理系統進行交互升級與安全機制優化。我們將把所有 CLI 數字選單全面改造為「鍵盤上下鍵高亮反白選擇」的互動介面。為防止資料誤刪，我們亦採用單一陣列狀態標籤技術，導入軟下架（置入資源回收桶）、還原重新上架、以及徹底清空（硬刪除）的防呆機制。

## Goals / Non-Goals

**Goals:**
- **終端機鍵盤控制選單**：實作通用鍵盤上下鍵監聽，結合 `_getch()` 與 ANSI 轉義字元 `\033[7m` 顯示高亮反白選項。
- **統一庫存與狀態標籤 (Status Tag)**：於多型 `MediaItem` 中新增 `status` 欄位（`"ACTIVE"` / `"ARCHIVED"`），統一資料庫以避免雙陣列 ID 衝突。
- **資源回收桶管理**：為管理員新增「回收桶與封存管理中心」子選單，實作下架封存（軟刪除）、重新上架（還原）及彻底清空（硬刪除）功能。
- **向下相容的載入解析**：資料庫載入邏輯必須具備容錯功能，當讀取到不含狀態欄位的舊資料時，預設自動標記為 `"ACTIVE"`。

**Non-Goals:**
- **建置雙重實體文字檔案**：不在硬碟建立 `archive_inventory.txt`，統一以 `inventory.txt` 儲存所有狀態之物件。
- **歷史交易紀錄狀態化**：`RentalRecord` 的生命週期維持原樣，軟刪除時不需要更改歷史交易的完整日誌。

## Decisions

### 1. 統一 Vector 與 Status 標籤設計
- **選擇技術**：全館庫存不論上架或封存，一律存在單一 `std::vector<std::shared_ptr<MediaItem>> inventory` 容器中。透過 `status` 變數篩選與區分。
- **替代方案**：使用在庫與封存兩個獨立的 Vector 陣列。
- **決策理由**：單一 Vector 確保了 ID 的唯一性，簡化了反序列化工廠代碼，寫入時也只要用單一 Loop 就可寫回單一 `inventory.txt` 檔案，降低複雜度。

### 2. 鍵盤事件攔截（_getch()）與 ANSI 反白控制
- **選擇技術**：結合 `<conio.h>` 的 `_getch()` 來捕捉方向鍵，使用 `\033[7m` 開啟 Reverse Video 渲染反白，`\033[0m` 重設。
- **替代方案**：使用 ncurses 或 Curses 等外部大型圖形終端庫。
- **決策理由**：本系統已在主程式設定了 `65001 (UTF-8)` 與 ANSI 控制支援，直接使用標準 ANSI 轉移序列沒有任何外部編譯依賴，最為乾淨與輕量。

## Risks / Trade-offs

- **[Risk] 舊庫存資料相容性問題**
  - **Trade-off**：現有的 `data/inventory.txt` 中不含 `,ACTIVE` 等狀態後綴，直接載入會造成欄位解析錯誤。
  - **Mitigation**：我們在 `LibrarySystem::loadData()` 中加入解析防禦：如果 split 出來的欄位數不足（比如少於 7 個），則自動將該多型物件的 `status` 設為 `"ACTIVE"`，完美相容舊資料。
- **[Risk] 鍵盤監聽特殊鍵干擾**
  - **Trade-off**：使用者在選單中按下 F1、Escape 或箭頭鍵以外的特殊鍵會產生不可期輸入。
  - **Mitigation**：在 `getMenuSelection` 中限定只接受方向鍵前置碼（0 或 224）與 Enter 鍵（13），其餘無效輸入一律忽略不動作。
