## 新增需求

### 需求：多型媒體表示
系統必須透過基底類別 `MediaItem` 與三個衍生類別 `Book`、`Dvd`、`Magazine` 建模媒體項目。每個具體類別都必須實作自己的虛擬函式 `getFee` 與 `serialize`。

#### 情境：顯示媒體型別專屬屬性
- **當** 系統顯示庫存項目
- **則** Book 必須顯示 ISBN 與作者，DVD 必須顯示導演與片長，Magazine 必須顯示期號與月份。

### 需求：媒體查找與篩選
系統必須允許使用者與管理者依唯一 ID、標題關鍵字或媒體類別（Book、DVD、Magazine）搜尋媒體。

#### 情境：以關鍵字搜尋媒體
- **當** 使用者輸入與媒體標題部分文字相符的關鍵字
- **則** 系統必須輸出所有符合條件的篩選結果。

### 需求：新增與移除媒體項目
系統必須允許管理者依唯一 ID 新增媒體項目或移除既有項目。

#### 情境：管理者新增一本書
- **當** 管理者輸入新書屬性（包含唯一 ID、標題、作者與 ISBN）
- **則** 系統必須建立 Book 實例、加入庫存向量，並將初始狀態設為未借出。
