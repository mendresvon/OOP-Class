# 測試與驗證

## 自動化測試入口

- PowerShell：`scripts/run-kv-tests.ps1`
- CMake target：`run_kv_tests`

此腳本包含三類驗證：

1. `kv-v1` 嚴格格式驗證（缺 header 啟動失敗）
2. 壞資料容錯驗證（壞行會輸出診斷並跳過，系統可啟動）
3. 密碼不落地明文驗證（`accounts.txt` 不含 `password=`）

## 核心流程驗證

- 註冊 / 登入 / 登出
- 借閱 / 歸還
- 管理者新增 / 下架 / 回收桶操作

## 資料格式驗證

- `kv-v1` 標頭存在性檢查
- 必要欄位缺失處理
- 無效資料列跳過與錯誤記錄
- 轉義值（`\|`, `\=`, `\\`, `\n`, `\r`）寫入/讀回一致

## 安全驗證

- 密碼不以明文寫入
- 雜湊比對流程正確
- 管理頁不顯示明文密碼

## 韌性驗證

- 模擬壞資料輸入不致使系統整體崩潰
- 原子儲存流程在中斷情境下不產生半套檔案

## 文件同步驗證

- 對照文件覆蓋矩陣確認本次必更文件
- DoD 檢核表全數勾選後才可結案
- 執行 `scripts/check-doc-sync.ps1`
- CI workflow `.github/workflows/doc-sync-gate.yml` 必須通過
